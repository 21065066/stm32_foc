# STM32 电机控制器通信协议测试指南

## 协议概述

本协议基于 UART 串口通信，用于上位机与 STM32 电机控制器之间的参数读写。

### 通信参数
- 波特率：115200
- 数据位：8
- 停止位：1
- 校验：无
- 字节序：小端（Little-Endian）

## 帧格式

### 通用帧结构（定长 20 字节）
| 字节偏移 | 长度    | 字段名       | 说明                              |
|----------|---------|--------------|-----------------------------------|
| 0        | 1 Byte  | SOF          | 帧头，固定 `0xAA`                 |
| 1        | 1 Byte  | CMD          | 命令字                            |
| 2        | 1 Byte  | PARAM_ID     | 参数 ID                           |
| 3        | 1 Byte  | DATA_LEN     | 数据段字节数（0~12）              |
| 4 ~ 15   | 12 Bytes| DATA         | 数据段（最大 12 字节，不足补 0）  |
| 16       | 1 Byte  | CHECKSUM     | 校验和                            |
| 17       | 1 Byte  | RESERVED     | 保留字节                          |
| 18       | 1 Byte  | RESERVED     | 保留字节                          |
| 19       | 1 Byte  | EOF          | 帧尾，固定 `0x55`                 |

### 说明
- 所有帧均为定长 20 字节
- DATA 段最大支持 12 个 uint8_t 数据
- DATA 段未使用的字节填充 0
- 保留字节（RESERVED）填充 0

## 命令字（CMD）

| CMD 值 | 名称             | 方向         | 说明                          |
|--------|------------------|--------------|-------------------------------|
| 0x01   | CMD_SET          | 上位机 → STM32 | 设置参数                      |
| 0x02   | CMD_GET          | 上位机 → STM32 | 请求读取参数                  |
| 0x81   | CMD_SET_ACK      | STM32 → 上位机 | 设置成功确认                  |
| 0x82   | CMD_GET_ACK      | STM32 → 上位机 | 读取参数确认（携带参数值）    |
| 0xFF   | CMD_ERROR        | STM32 → 上位机 | 错误响应                      |

## 参数 ID 表（PARAM_ID）

所有参数值均以 IEEE 754 单精度浮点（float，4字节，小端）传输。

### 访问权限说明
| 符号 | 含义       |
|------|------------|
| R    | 只读       |
| W    | 只写       |
| RW   | 可读可写   |

## 交互流程

### 1. 设置参数（写）
```
上位机 → STM32:  AA 01 PARAM_ID 04 [float 4字节] CHECKSUM 55
```

### 2. 读取参数（读）
```
上位机 → STM32:  AA 02 PARAM_ID 00 CHECKSUM 55
STM32  → 上位机:  AA 82 PARAM_ID 04 [float 4字节] CHECKSUM 55
```

## 校验和计算规则

```
CHECKSUM = (CMD + PARAM_ID + DATA_LEN + DATA[0] + ... + DATA[11]) & 0xFF
```

注意：SOF（0xAA）和 EOF（0x55）不参与校验计算，DATA 段固定计算 12 个字节（包括填充的 0）。

## 错误码（ERR_CODE，1 Byte）

| 错误码 | 含义                       |
|--------|----------------------------|
| 0x01   | 未知参数 ID                |
| 0x02   | 数据长度错误               |
| 0x03   | 校验和错误                 |
| 0x04   | 参数值超出范围             |
| 0x05   | 帧格式错误（SOF/EOF 异常）|

## 测试示例

### 示例1：读取极对数（PARAM_ID = 0x01）
发送帧（20 字节）：`AA 02 01 00 00 00 00 00 00 00 00 00 00 00 00 00 02 00 00 55`
- SOF: 0xAA
- CMD: 0x02 (CMD_GET)
- PARAM_ID: 0x01
- DATA_LEN: 0x00
- DATA: 12 个 0x00（填充）
- CHECKSUM: 0x03 (0x02 + 0x01 + 0x00 + 0x00×12)
- RESERVED: 0x00, 0x00
- EOF: 0x55

预期响应（20 字节）：`AA 82 01 04 00 00 00 00 00 00 00 00 00 00 00 00 07 00 00 55`
- SOF: 0xAA
- CMD: 0x82 (CMD_GET_ACK)
- PARAM_ID: 0x01
- DATA_LEN: 0x04（返回 4 字节数据）
- DATA: 4 字节 float 值 + 8 字节填充 0
- CHECKSUM: 校验和
- RESERVED: 0x00, 0x00
- EOF: 0x55

### 示例2：设置Speed PID Kp（PARAM_ID = 0x20，值为2.5）
发送帧（20 字节）：`AA 01 20 04 00 00 20 40 00 00 00 00 00 00 00 00 65 00 00 55`
- SOF: 0xAA
- CMD: 0x01 (CMD_SET)
- PARAM_ID: 0x20
- DATA_LEN: 0x04（4 字节数据）
- DATA: 0x00 0x00 0x20 0x40 (float 2.5 的小端表示) + 8 字节填充 0
- CHECKSUM: 0x65 (0x01 + 0x20 + 0x04 + 0x00 + 0x00 + 0x20 + 0x40 + 0x00×8)
- RESERVED: 0x00, 0x00
- EOF: 0x55

预期响应（20 字节）：`AA 81 20 00 00 00 00 00 00 00 00 00 00 00 00 00 A1 00 00 55`
- SOF: 0xAA
- CMD: 0x81 (CMD_SET_ACK)
- PARAM_ID: 0x20
- DATA_LEN: 0x00
- DATA: 12 字节填充 0
- CHECKSUM: 0xA1 (0x81 + 0x20 + 0x00 + 0x00×12)
- RESERVED: 0x00, 0x00
- EOF: 0x55

## 注意事项

1. 所有帧均为定长 20 字节
2. DATA 段最大支持 12 个 uint8_t 数据，不足时用 0 填充
3. 所有浮点数使用 IEEE 754 单精度格式（4字节）
4. 字节序为小端（Little-Endian）
5. 每次通信都会收到 ACK 确认或错误响应
6. 校验和计算包括完整的 12 字节 DATA 段（包括填充的 0）
7. 协议使用空闲中断检测帧结束
